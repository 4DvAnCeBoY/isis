What next?

Decide which viewer(s) you want to provide for accessing your domain objects, and then comment out/delete the relevant entries from the WEB-INF/web.xml and delete the static resources that you no longer need. You'll probably also want to delete this file, and remove the <filter>RedirectToDocsFilter</filter> entry in web.xml.

Then, decide which objectstore you want to use. While doing initial prototyping you may find that the default in-memory objectstore suffices; the isis.fixtures property in the WEB-INF/isis.properties configuration file the initial data set that is used to seed the database. If you decide to use the NoSQL or SQL databases, follow the more detailed comments in the WEB-INF/isis.properties config file.

Finally, don't forget that there is more information at the Apache Isis website, and more help on the Apache Isis mailing lists (subscribe here, post here)
