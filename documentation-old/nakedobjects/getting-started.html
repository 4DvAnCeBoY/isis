Getting started

Creating an Exploration application

To create a exploration application create a new class extending Exploration, add a setUp method to specify what classes to make available to the user, and add a main method to start it.  If you want to create some intial objects these can be added in the setup method.  The following serves as a simple example of all the above aspects.

import org.nakedobjects.Exploration;

public class ExpensesExploration extends Exploration {
	public void setUp() {
		registerClass(Employee.class);
		registerClass(Project.class);
		registerClass(Expense.class);
		registerClass(Claim.class);


		if(hasNoInstances(Employee.class)) {
 			Employee newUser;
			
			newUser = (Employee) createInstance(Employee.class);
			newUser.getFirstName().setValue("John");
			newUser.getSurname().setValue("Smith");
		}	
	}

	public static void main(String[] args) {
		new ExpensesExploration();
	}
}

Creating a Executable Acceptance Test (XAT)

To create an acceptance test (a Junit test designed to test the functionality of a Naked Objects application) you must extend AcceptanceTestCase.  Again classes that the user has direct access to must be registered and this is similarly done in the setUp method.

import org.nakedobjects.utility.ComponentException;
import org.nakedobjects.utility.ConfigurationException;
import org.nakedobjects.xat.AcceptanceTestCase;

public class ECSAcceptanceTest extends AcceptanceTestCase {

    public void setUp() throws ConfigurationException, ComponentException {
        super.setUp();
        
        registerClass(Booking.class);
        registerClass(City.class);
        registerClass(Customer.class);
        registerClass(Location.class);
        registerClass(CreditCard.class);
        registerClass(Telephone.class);
    }
}
