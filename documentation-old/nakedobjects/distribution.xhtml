Distribution of Naked Objects accross tiers

When Naked Objects works in a multitiered environment client requests to the reflection mechanism, class manager and object manager need to be passed across to the server for execution. This requires new object and class managers, and for the reflection classes to be wrapped so that they all forward the necessary request to the server for execution and deal with its responses.

The following details all the request that are made by a remote client:-

  • destroyObject() passes accross the object's, to be destroyed, OID. There is no normal response, however if the object store cannot comply an Exception will be thrown.
  • getInstances() passes across the name of a class, a class and a text pattern, or a pattern object. It recieves an array of OID to the set of ...
  • hasInstance() passes across the name of a class, and

The following are all ignored; they are not forwarded to the server:-

NakedObjectManager.createOid() throws an exception; it shoul never be called by the client

NakedObjectManager.abortTransaction()

NakedObjectManager.startTransaction()

NakedObjectManager.endTransaction()

Notes

It is extremely important that only one instance of any object extists, that is an object having a unique OID. To ensure this does not occur the 'loaded objects' set should be checked, using the OID, before adding a new object to the system. If it already exists then the existing object can be retrieved from this set using the OID.
