Project Structure

Naked Objects is broken up in a number of sub-projects, each of which deals with a specific aspect of  the framework.  The central project is no-core, and

Structure of a sub-project

Each sub project always has a source, test and example directories, for the framework code, the unit tests, and usage examples respectively.  If needed there will be directories for library files, external scripts and executables, documentation, and notes.  In addition, as part of the build a build directory, and as part of running within eclipse, a classes directory.

The structure of the source and test directorys should be the same so that the tests are in a separate, but parallel tree.

Files included in the project directory include the Eclipse project configuration files, an Ant build configuration, a read me.

The project directory will look as follows (square brackets indicate optional directories, curly brackets indicate those generated by the build on the IDE):-

 - [bin]
 - {build}
 - {classes}
 - [doc]
 - example
 - [lib]
 - [notes]
 - src
 - test
 - .classpath
 - .project
 - build.xml
 - README
