Naked Object Extensible Distribution Mechanism

Client

To set up a distribution mechanism for Naked Objects you need to create implementations of the following interfaces, each which represents a type of data that needs to be passed between the client and server:-

  • AboutData
  • ActionType
  • ClassReference
  • InstanceSet
  • ObjectData
  • ObjectReference
  • ObjectUpdateMessage
  • OidData
  • ParameterSet
  • SecurityToken

Also implement a factory (implementing the RemoteObjectFactory interface) to generate each of the above data types.

Finally the requests from the Naked Objects client need to transfered to the server through an implementation of the DistributionInterface interface. Each method that is implemented makes use of the above data objects so you are in complete control over what kind of data get passed over the network. Note that some of the methods throw exceptions, and these need to be propogated over the network and rethrown by the client when necessary.

Server

The server must receive the requests from the client and then call the corresponding method on the RemoteMechanism object to execute the request on the server. Any response, or any thrown exception must be passed back to the client.
